# PromptSack

**v1.0.0** — Encrypted local prompt catalog with hybrid semantic search.

PromptSack is a desktop app for creating, organizing, locking, and **searching** prompts. Content lives as plain Markdown files. Search and catalog metadata live in an encrypted **MongrelDB** index. Embeddings are **all-MiniLM-L6-v2** (384-d) by default, with optional OpenAI-compatible remote embeddings.

### Architecture

| Layer | Role |
|---|---|
| **Tauri webview** | Renders `ui/`; talks to Rust via `window.promptsack` → `invoke` |
| **`bridge.js`** | Thin IPC adapter (same API shape as the old Electron preload) |
| **`commands.rs`** | Rust command surface: vault, prompts, folders, search, settings… |
| **`vault/store.rs`** | Orchestrates Markdown FS + MongrelDB index + session lock |
| **`vault/markdown.rs`** | `prompts/{id}.md` frontmatter + body |
| **`vault/schema.rs`** | MongrelDB tables: `prompt_index`, `folders`, `smart_folders`, `meta`… |
| **`vault/embed.rs`** | fastembed MiniLM or OpenAI-compatible HTTP |
| **`vault/crypto.rs`** | App-level AES-GCM for locked prompt bodies |
| **OS keyring** | Session lock password + embedding API keys |

**Write path:** write `.md` → embed title/body/notes → upsert `prompt_index`.  
**Search path:** embed query → ANN (`ann_search`) + lexical hybrid.  
**Open path:** rebuild `prompt_index` from Markdown (MongrelDB 0.60.x cold-open embedding workaround).

Vault on disk:

```
$PROMPTSACK_DATA_DIR  or  <app data>/vault/
  prompts/{uuid}.md
  index.mdb/          # encrypted MongrelDB
```

## Features

- Three-pane catalog UI (sidebar / list / editor)
- Semantic + lexical hybrid search (MiniLM 384-d)
- Folders, **smart folders** (saved filters), tags, favorites, locks
- Bulk select: move / export / delete
- Import / export JSON; native file dialogs
- Dual encryption: MongrelDB at rest + optional per-prompt locks
- Pluggable embeddings: local MiniLM (default) or OpenAI-compatible API

## Quick start

```bash
git clone https://github.com/visorcraft/promptsack.git
cd promptsack
npm install
npm test
npm start          # tauri dev
```

On first launch, choose a **vault passphrase**. Override vault location with:

```bash
export PROMPTSACK_DATA_DIR=/path/to/vault
```

On some Linux Wayland setups WebKit is unstable; prefer X11 if the window fails to open:

```bash
GDK_BACKEND=x11 WEBKIT_DISABLE_DMABUF_RENDERER=1 npm start
```

## Development map

### Frontend

1. Edit `src/renderer/*` and `src/shared/*`
2. `npm run ui:build` (or let `tauri dev` run `beforeDevCommand`)
3. UI loads `ui/bridge.js` then `ui/app.js`

IPC names are snake_case Rust commands (`vault_open`, `prompts_search`, …). The bridge maps camelCase JS args.

### Backend

1. Edit `src-tauri/src/vault/*` or `commands.rs`
2. `cargo test --manifest-path src-tauri/Cargo.toml`
3. `tauri dev` hot-reloads the Rust side

### Tests

- **Rust** (`src-tauri`): vault CRUD, lock, reopen, search, import/export, crypto, markdown
- **JS** (`tests/`): repo contract + save-policy regressions (no Electron / Node MongrelDB)

## Encryption & secrets

| Layer | Protects | Mechanism |
|---|---|---|
| MongrelDB at rest | Index (vectors, FTS, folders, meta) | AES-256-GCM · vault passphrase |
| App locks | Locked prompt body/notes in Markdown | AES-GCM · session password |
| API keys | OpenAI-compatible credentials | OS keychain (`keyring`) |
| Session password | Auto-unlock preference | OS keychain |

## Embedding providers

| Provider | Default | Notes |
|---|---|---|
| `local-minilm` | ✅ | fastembed `AllMiniLML6V2` · offline after first model download |
| `openai-compatible` | optional | Any `/v1/embeddings` · force `dimensions: 384` · key in keychain |

Configure under **Settings** in the app.
